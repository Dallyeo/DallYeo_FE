## P1 — 둘러보는 방문자 (비로그인)

| 항목 | 내용 |
|---|---|
| **상황** | 앱을 설치/실행했지만 아직 로그인하지 않음. 군산 추천 코스를 자유롭게 둘러봄. |
| **목표** | 부담 없이 추천 코스를 살펴보고, 마음에 들면 달리기를 시작하거나 가입을 결정. |
| **행동 패턴** | 메인뷰 탐색 → 코스 카드/ i-버튼으로 상세 확인 → 기록 탭/마이페이지 개인 기능 시도 시 로그인 요구를 마주침. |
| **니즈** | 로그인 강요 없이 핵심 콘텐츠(추천 코스)를 미리 경험. 로그인이 필요한 지점은 명확하고 비파괴적이어야 함. |
| **페인 포인트** | 갑작스러운 풀스크린 로그인 차단, 둘러보기 흐름 단절, 왜 로그인이 필요한지 모호함. |
| **핵심 제약 (출처)** | 게이트 액션 시 바텀시트 자동 노출(FR-LOGIN-02/03), 마이페이지 상단 배너(FR-LOGIN-04), 기록 탭 차단(FR-V02-03), 마이페이지 일부 비활성(FR-V02-04). |

---

## P2 — 달리는 회원 (로그인)

| 항목 | 내용 |
|---|---|
| **상황** | 네이티브 OAuth로 로그인 완료. `AppSession` 토큰을 브릿지로 주입받은 상태. |
| **목표** | 추천 코스를 이용하고, 자신의 기록/마이페이지 등 개인화 기능에 자유롭게 접근. |
| **행동 패턴** | 메인뷰 → 기록 탭 자유 진입 → 마이페이지 프로필/계정 관리 → 세션 만료/로그아웃 시 적절한 안내를 기대. |
| **니즈** | 인증 상태가 모든 화면에서 일관되게 반영. 세션이 끊겼을 때 데이터 손실 없이 명확한 안내. |
| **페인 포인트** | 화면마다 다른 인증 상태, 세션 만료 시 무음 실패(아무 안내 없는 빈 화면), 로그아웃 경로 불명확. |
| **핵심 제약 (출처)** | 세션 단일 출처=네이티브(NFR-AUTH-02), `sessionChanged` 단일 구독(NFR-AUTH-03), 세션 무효화 시 V02 라우팅+토스트(FR-LOGIN-06), 기록/마이페이지 자유 진입(FR-V02-03/04). |

## P3 — 처음 켠 사용자 (온보딩)

| 항목 | 내용 |
|---|---|
| **상황** | 앱 최초 실행. 온보딩 플로우를 1회 경험 (서비스 소개 → 위치 권한 → 기본 정보). |
| **목표** | 빠르게 본 화면(V02)에 도달. 위치 권한과 신체 정보는 부담 없이(거부/건너뛰기 허용) 처리. |
| **행동 패턴** | 서비스 소개 확인 → 위치 권한 허용/거부 → 키·체중·성별 입력 또는 건너뛰기 → 메인뷰 진입. 이후 재실행 시 온보딩 미표시. |
| **니즈** | 강제 입력 없이 진행 가능. 입력 시에는 잘못된 값(자릿수 초과 등)이 즉시 차단되어 실수 방지. |
| **페인 포인트** | 권한 거부 시 막다른 길, 신체 정보 강제, 잘못된 입력에 대한 늦은 피드백, 온보딩 반복 노출. |
| **핵심 제약 (출처)** | 1회성 표시(FR-V01-01), 위치 권한 거부해도 진행(FR-V01-03), 키≤3자리/체중 2~3자리 검증(FR-V01-05/06), 4-옵션 성별(FR-V01-07), 건너뛰기 허용(FR-V01-08), 3항목 충족 시에만 "입력 완료" 활성(FR-V01-09). |
