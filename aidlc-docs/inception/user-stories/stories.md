# 🔐 EPIC-LOGIN — 로그인

> 비로그인 사용자가 게이트된 기능에 도달했을 때 자연스럽게 로그인하고, 세션이 끊기면 안전하게 복귀하는 흐름.

---

### LOGIN-S1 — 게이트 액션 시 로그인 바텀시트 자동 노출
**As a** 둘러보는 방문자(P1)
**I want** 로그인이 필요한 기능을 누르면 화면을 떠나지 않고 하단에서 로그인 안내가 뜨길
**So that** 둘러보던 맥락을 잃지 않고 로그인 여부를 결정할 수 있다.

- **페르소나**: P1 (주), P3 (온보딩 직후 합류)
- **태그**: FR-LOGIN-02, FR-LOGIN-03 · SECURITY-08(접근 제어) · NFR-WEBVIEW-04(history 관리) · PBT 후보: 게이트 액션 집합 → 항상 바텀시트 1개만 오픈(idempotent)

**수용기준**
```gherkin
Given 비로그인 사용자가 메인뷰에 있고
When 게이트된 액션(기록 탭 / 마이페이지 프로필·내정보수정·계정관리 / V10 완주결과 저장)을 시도하면
Then 현재 화면 위에 로그인 바텀시트가 표시되고
And 기저 화면(메인뷰)은 그대로 유지된다 (풀스크린 전환·라우트 이동 없음).

Given 로그인 바텀시트가 열려 있고
When 사용자가 좌측 스와이프(iOS 뒤로) 또는 시트 바깥을 탭하면
Then 바텀시트는 닫히고 직전 화면 상태가 보존된다 (history 엔트리로 관리).

Given 이미 로그인 바텀시트가 열린 상태에서
When 또 다른 게이트 액션이 트리거되면
Then 중복 시트가 쌓이지 않고 단일 시트만 유지된다.
```

### LOGIN-S2 — Kakao / Apple 소셜 로그인
**As a** 둘러보는 방문자(P1)
**I want** Kakao 또는 Apple 버튼으로 로그인하길
**So that** 별도 가입 절차 없이 회원 기능을 쓸 수 있다.

- **페르소나**: P1 → P2 전환
- **태그**: FR-LOGIN-01, FR-LOGIN-05 · SECURITY-12(인증·자격증명) · NFR-AUTH-01(OAuth 위임), NFR-AUTH-02(토큰 단일 출처=네이티브) · NFR-BRIDGE-02(request-id promise) · PBT 후보: 로그인 응답 상태(성공/취소/실패) 처리 분기

**수용기준**
```gherkin
Given 로그인 바텀시트가 열려 있고
When 사용자가 본다면
Then 제공자 버튼은 Kakao, Apple 두 개만 노출되고 Google 버튼은 보이지 않는다.

Given 사용자가 Kakao(또는 Apple) 버튼을 탭하면
When 웹은 OAuth 핸드셰이크를 직접 수행하지 않고 오직 bridge.login(provider)만 호출하고
Then 네이티브가 핸드셰이크를 수행한 뒤 AppSession을 반환하며
And 웹은 토큰을 메모리에만 보관(localStorage/cookie/IndexedDB 저장 금지)하고 바텀시트를 닫는다.

Given 사용자가 네이티브 로그인 화면에서 취소하면
When bridge.login이 취소로 resolve/reject되면
Then 바텀시트는 열린 상태로 유지되고 에러 토스트 없이 재시도가 가능하다.

Given 로그인 처리 중 네트워크/제공자 오류가 발생하면
When bridge.login이 실패를 반환하면
Then "로그인에 실패했어요. 다시 시도해 주세요" 안내를 표시하고 토큰 상태는 변경되지 않는다.
```

### LOGIN-S3 — 마이페이지 상단 "로그인하세요" 배너
**As a** 둘러보는 방문자(P1)
**I want** 마이페이지 상단에서 로그인 안내 배너를 보길
**So that** 어디서 로그인하는지 한눈에 알고 바로 진입할 수 있다.

- **페르소나**: P1
- **태그**: FR-LOGIN-04, FR-V02-04 · SECURITY-08 · NFR-UI-02(한국어 카피)

**수용기준**
```gherkin
Given 비로그인 사용자가 마이페이지(V13)에 진입하면
When 화면이 렌더링되면
Then 상단에 "로그인하세요" 배너가 표시된다.

Given "로그인하세요" 배너가 보이고
When 사용자가 배너를 탭하면
Then LOGIN-S1의 로그인 바텀시트가 열린다.

Given 사용자가 로그인에 성공하면
When 마이페이지가 다시 렌더링되면
Then "로그인하세요" 배너는 더 이상 표시되지 않는다.
```

### LOGIN-S4 — 세션 만료 / 로그아웃 처리
**As a** 달리는 회원(P2)
**I want** 세션이 만료되거나 로그아웃되면 명확한 안내와 함께 안전한 화면으로 이동하길
**So that** 무음 실패나 빈 화면 없이 상황을 이해할 수 있다.

- **페르소나**: P2
- **태그**: FR-LOGIN-06 · SECURITY-11(보안 핵심 로직 분리), SECURITY-12 · NFR-AUTH-03(sessionChanged 단일 구독) · PBT 후보: 세션 상태 머신(로그인↔로그아웃↔만료) 전이 불변식

**수용기준**
```gherkin
Given 로그인 사용자가 임의 화면에 있고
When API가 401을 반환하거나 네이티브 sessionChanged 이벤트가 logout 상태로 도착하면
Then 클라이언트 세션 상태가 초기화되고
And V02 메인뷰로 라우팅되며
And "로그아웃되었습니다" 토스트가 1회 표시된다.

Given 짧은 시간에 401 응답과 sessionChanged 이벤트가 동시에 발생하면
When 두 신호가 모두 도착해도
Then 세션 초기화·라우팅·토스트는 중복 없이 단 한 번만 수행된다.

Given 세션이 초기화된 직후
When 사용자가 메인뷰에 있으면
Then 비로그인 게이트 규칙(LOGIN-S1, FR-V02-03/04)이 즉시 적용된다.
```

# 🧭 EPIC-V01 — 온보딩

> 최초 실행 시 1회만 노출되는 진입 플로우. 권한·신체 정보는 부담 없이(거부/건너뛰기 허용).

---

### V01-S1 — 최초 실행 시 온보딩 1회 표시
**As a** 처음 켠 사용자(P3)
**I want** 앱을 처음 열 때만 온보딩을 보길
**So that** 매번 반복되지 않고 빠르게 본 화면에 도달한다.

- **페르소나**: P3
- **태그**: FR-V01-01, FR-V01-02 · NFR-WEBVIEW-04 · PBT 후보: "완료 또는 건너뛰기 → 재표시 안 됨" 멱등 플래그

**수용기준**
```gherkin
Given 앱을 최초 실행한 사용자가
When 앱에 진입하면
Then 온보딩 플로우(① 서비스 소개 → ② 위치 권한 안내 → ③ 기본 정보 입력)가 순서대로 표시된다.

Given 사용자가 온보딩을 완료하거나 건너뛰면
When 이후 앱을 다시 실행하면
Then 온보딩은 다시 표시되지 않고 V02 메인뷰로 바로 진입한다.

Given 온보딩 완료 플래그가 존재하면 (신체 정보 변경은 V13 설정에서만 가능)
When 사용자가 다시 신체 정보를 바꾸려 하면
Then 온보딩 재진입이 아니라 설정(V13) 경로로 안내된다. (V13 본체는 본 라운드 범위 밖)
```

### V01-S2 — 위치 권한 요청 단계
**As a** 처음 켠 사용자(P3)
**I want** 위치 권한을 요청받되 거부해도 계속 진행할 수 있길
**So that** 권한 결정이 막다른 길이 되지 않는다.

- **페르소나**: P3
- **태그**: FR-V01-03 · NFR-BRIDGE-01/02(브릿지 추상화·async) · NFR-BRIDGE-03(mock bridge) · PBT 후보: 권한 상태(granted/denied/blocked) → 다음 단계 진행 가능성 불변

**수용기준**
```gherkin
Given 사용자가 위치 권한 단계에 도달하면
When 권한 요청 버튼을 탭하면
Then bridge.requestPermission('location')이 호출된다.

Given 사용자가 권한을 허용하면
When 권한 상태가 granted로 반환되면
Then 다음 단계(기본 정보 입력)로 진행한다.

Given 사용자가 권한을 거부하거나 차단(blocked)하면
When 권한 상태가 denied/blocked로 반환되면
Then 제한되는 기능에 대한 안내를 표시하되 다음 단계로 계속 진행할 수 있다.

Given 브라우저 단독(디바이스 없음) 환경에서
When mock bridge가 권한 결과를 비동기로 반환하면
Then 실제 디바이스와 동일하게 단계 진행 흐름이 동작한다.
```

### V01-S3 — 기본 정보 입력 (키 / 체중 / 성별) + 검증
**As a** 처음 켠 사용자(P3)
**I want** 키·체중·성별을 입력하되 잘못된 값은 즉시 막아주길
**So that** 실수 없이 정확한 신체 정보를 남길 수 있다.

- **페르소나**: P3
- **태그**: FR-V01-04, FR-V01-05, FR-V01-06, FR-V01-07, FR-V01-09 · SECURITY-05(입력 검증) · PBT 후보: 키/체중 입력값 도메인 검증(자릿수·정수) property, "3항목 충족 ⇔ 버튼 활성" 동치

**수용기준**
```gherkin
Given 기본 정보 입력 화면에서
When 사용자가 키를 입력하면
Then 정수만 허용되고 최대 3자리까지만 입력되며 4자리 이상·비정수는 차단된다.

Given 기본 정보 입력 화면에서
When 사용자가 체중을 입력하면
Then 정수만 허용되고 2~3자리만 유효하며 1자리·4자리 이상은 차단된다.

Given 성별 입력에서
When 사용자가 선택하면
Then 남 / 여 / 기타 / 선택안함 4개 중 정확히 하나만 선택된다.

Given 키·체중·성별 중 하나라도 비어 있으면
When 화면을 보면
Then "입력 완료" 버튼은 비활성 상태이다.

Given 키·체중·성별 3개 항목이 모두 유효하게 채워지면
When 사용자가 "입력 완료"를 탭하면
Then 버튼이 활성화되어 있고 탭 시 V02 메인뷰로 이동한다.
```

# 🏠 EPIC-V02 — 메인뷰

> 모든 사용자의 홈 허브. 추천 코스 탐색 + 탭바 네비게이션 + 네이티브 코스 플로우 진입.

---

### V02-S1 — 메인뷰 진입 & 추천 코스 리스트
**As a** 둘러보는 방문자(P1)
**I want** 메인 화면에서 군산 추천 코스 카드를 보길
**So that** 어떤 코스가 있는지 빠르게 탐색할 수 있다.

- **페르소나**: P1, P2
- **태그**: FR-V02-01, FR-V02-08 · NFR-DATA-01(MSW/실 백엔드 혼용), NFR-DATA-02(region 확장 가능) · SECURITY-15(예외처리) · PBT 후보: 코스 리스트 렌더(0개/N개) 불변식

**수용기준**
```gherkin
Given 사용자가 메인뷰에 진입하면
When 추천 코스 데이터가 정상 로드되면
Then 추천 코스 카드 리스트, 지역 선택기, "코스 만들기" 버튼, 하단 탭바가 표시된다.

Given 추천 코스 목록이 비어 있으면 (빈 응답)
When 메인뷰가 렌더링되면
Then 빈 상태(Empty) 안내가 표시되고 다른 UI(탭바·코스 만들기)는 정상 동작한다.

Given 추천 코스 API 호출이 실패하면 (네트워크/서버 오류)
When 메인뷰가 렌더링되면
Then 에러 상태와 재시도 수단이 표시된다.
```

### V02-S2 — 하단 탭바 네비게이션 & 게이트
**As a** 둘러보는 방문자(P1) / 달리는 회원(P2)
**I want** 메인 / 기록 / 마이페이지 탭으로 이동하되 인증 상태에 맞게 동작하길
**So that** 권한 없는 기능은 막히고 가능한 기능은 자유롭게 쓸 수 있다.

- **페르소나**: P1, P2
- **태그**: FR-V02-02, FR-V02-03, FR-V02-04 · SECURITY-08(접근 제어), SECURITY-11 · NFR-AUTH-03 · PBT 후보: (인증상태 × 탭) → 허용/차단 결정표 전수 검증

**수용기준**
```gherkin
Given 어떤 메인 화면이든
When 하단 탭바가 표시되면
Then 메인 / 기록 / 마이페이지 3개 탭만 있고 업적 탭은 없다.

Given 비로그인 사용자가
When "기록" 탭을 탭하면
Then V11으로 이동하지 않고 로그인 바텀시트(LOGIN-S1)가 열린다.

Given 로그인 사용자가
When "기록" 탭을 탭하면
Then V11 기록뷰로 이동한다.

Given 비로그인 사용자가
When "마이페이지" 탭을 탭하면
Then V13 설정뷰로 진입하되 상단에 "로그인하세요" 배너(LOGIN-S3)가 보이고
And 프로필 카드 / 내정보수정 / 계정관리 항목은 비활성화되어 탭 시 로그인 바텀시트가 열린다.

Given 로그인 사용자가
When "마이페이지" 탭을 탭하면
Then V13 설정뷰에 자유롭게 진입하고 배너는 표시되지 않는다.
```

### V02-S3 — 코스 만들기 / 코스 확인 (네이티브 진입)
**As a** 둘러보는 방문자(P1)
**I want** 코스를 직접 만들거나 추천 코스를 눌러 확인 화면으로 가길
**So that** 달리기를 준비하는 다음 단계로 매끄럽게 이어진다.

- **페르소나**: P1, P2
- **태그**: FR-V02-05, FR-V02-06 · NFR-BRIDGE-01/02 · PBT 후보: 코스 카드 탭 → 정확히 1회 openCourseConfirm(course) 호출

**수용기준**
```gherkin
Given 사용자가 메인뷰에 있고
When "코스 만들기" 버튼을 탭하면
Then bridge.openCourseSearch()가 호출되어 네이티브 검색뷰(V04)로 전환된다.

Given 사용자가 추천 코스 카드 본문 영역을 탭하면
When 탭이 인식되면
Then bridge.openCourseConfirm(course)가 해당 코스 데이터와 함께 호출되어 네이티브 코스 확인뷰(V08)로 전환된다.

Given 브릿지가 사용 불가능한 환경(mock bridge)에서
When 위 버튼/카드를 탭하면
Then mock bridge가 호출을 기록하고 브라우저에서 흐름이 깨지지 않는다.
```

### V02-S4 — 지역 선택기 (군산 한정, 확장 가능)
**As a** 둘러보는 방문자(P1)
**I want** 현재 지역(군산)을 확인하고 선택 UI를 보길
**So that** 추천 코스가 어느 지역 기준인지 알 수 있다.

- **페르소나**: P1, P2
- **태그**: FR-V02-08 · NFR-DATA-02(region 하드코딩 금지·동적 조회) · PBT 후보: 지원 지역 목록(1개~N개) → 기본 선택 항상 유효

**수용기준**
```gherkin
Given 사용자가 메인뷰에 진입하면
When 지역 선택기가 렌더링되면
Then 기본 선택값은 "군산"이다.

Given MVP1 환경에서
When 사용자가 지역 선택기를 열면
Then 선택 UI는 노출되지만 실제 선택 가능한 지역은 군산뿐이다 (미래 확장 자리만 마련).

Given region 데이터 모델이
When 코드에 정의되면
Then "군산"은 enum/literal로 하드코딩되지 않고 regionCode(string) + 동적 지역 목록 조회 구조를 따른다.
```

### V02-S5 — 추천 코스 i-버튼 정적 미리보기 팝업
**As a** 둘러보는 방문자(P1)
**I want** 코스 카드의 i-버튼으로 설명과 경로 미리보기를 보길
**So that** 코스 확인 화면에 진입하기 전에 내용을 가늠할 수 있다.

- **페르소나**: P1, P2
- **태그**: FR-V02-07 · NFR-WEBVIEW-04(모달=history 엔트리) · SECURITY-15(이미지 로드 실패 처리) · PBT 후보: 팝업 열기/닫기 토글 멱등

**수용기준**
```gherkin
Given 사용자가 추천 코스 카드의 i-버튼을 탭하면
When 팝업이 열리면
Then 코스 설명과 정적 경로 미리보기 이미지가 표시되고 지도 SDK는 사용되지 않는다.

Given i-버튼 팝업이 열려 있고
When 사용자가 좌측 스와이프(뒤로) 또는 닫기를 하면
Then 팝업이 닫히고 메인뷰 상태가 보존된다 (history 엔트리 관리).

Given 정적 미리보기 이미지 로드가 실패하면
When 팝업이 표시되면
Then 이미지 영역에 대체(플레이스홀더) 표시가 나타나고 설명 텍스트는 정상 표시된다.
```
